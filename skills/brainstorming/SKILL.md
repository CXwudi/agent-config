# Brainstorming Skill

## Steps

1. Start from exploring project context
2. Brainstorming with user by asking user quesions.
    1. Ask one question at a time
3. When you have enough context and clarification, present your ideas or solution to the user.
    - If there are multiple viable options, present the recommended option, and slightly mention the other options, and explain why you recommend the option.

## Notes

- Prefer multiple choice questions when possible, but use open-ended questions when the user is still shaping the idea
- When asking questions, cover scope, constraints, success criteria, and risks.
- When explaining, you can also use ASCII diagrams, or even write down your Mermaid diagrams in a markdown file to help visualize your ideas
