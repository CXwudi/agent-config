# Brainstorming Skill

## Steps

1. Start from exploring project context, documentation, espacially those related to the topic, idea, or problem.
    1. If you cannot find enough relevant context, ask the user where to find it.
2. Brainstorming with user by asking quesions.
    1. Ask one question at a time.
3. When you have enough context and clarification, present your ideas or solution to the user.
    - If there are multiple viable options, present the recommended option, and briefly mention the other options, and explain why you recommend the option.
4. The skill is done when the user has reached the desired clarity or chosen a direction.

## Notes

- Prefer multiple choice questions when possible, but use open-ended questions when the user is still shaping the idea
- If the topic related to coding, you can ask questions related to scope, constraints, success criteria, and risks.
- Do not ask questions that are already answered from the project context.
- When explaining, you can also use ASCII diagrams, or even write down your Mermaid diagrams in a markdown file to help visualize your ideas.
